# 递归式问答系统设计史 v12b：选择性吸收 v8 的可执行 / 可校准子集

## 1. 文档定位

本文件不重新发明 v8，也不宣称 v8 已整体落地。

它要说明的是：在当前代码、合同与测试能力边界下，系统为什么只先吸收 v8 中**可执行、可校准、可被当前主线约束的那一部分**，而不把整套“生成式世界引擎”一次性压进当前版本。

一句话：

```text
当前主线吸收的是 v8 中能进入现有 Episode / 合同 / 审计 / 未来测试闭环的子集，
而不是提前宣称系统已经拥有完整 predictive / interventional world engine。
```

---

## 2. 为什么不是整体吸收 v8

v8 的完整主张包含四个连续层次：

```text
观测投影化
→ 机制程序化
→ 反事实生成
→ 实验设计与动作闭环
→ 预测误差校准本体
```

但根据 `docs/current/testing-roadmap-v7-to-v11.md`，当前现实是：

- v7 主链路已跑通
- v8 处于“类型与存储已落地，但占位符为主”的阶段
- v8 真正需要的验证环境是“可控、可重置、可观测的仿真世界”
- 该仿真与校准环境仍属于 Phase 2 / Phase 3 的建设目标，而不是已完成能力

因此当前不能把 v8 当作“已经形成完整预测引擎”的事实层能力写入设计史，而只能把它拆成两类：

- 可先吸收的：已经有对象锚点、能接入当前审计链、将来能被测试路线图继续推进的部分
- 暂不吸收的：一旦写成当前能力，就会把“对象壳”误写成“世界引擎”的部分

这不是降级 v8，而是避免把设计上限和当前可校准能力混写。

---

## 3. 当前先吸收的是什么

当前主线先吸收的是 v8 里最靠近“描述性重建之后的最小可执行接口”的部分。

### 3.1 已有明确锚点的部分

根据当前合同与路线图，至少有以下锚点已经成立或进入了足够明确的对象边界：

1. `ObservationModel`

- `observation-model-contract.md` 已是 `current`
- `ObservationRecord.observationModelId` 已作为主线绑定存在
- 系统已经明确承认“观测是投影，不是事实本身”

2. `ActionExecution`

- `action-execution-contract.md` 已是 `current`
- `ExperimentDesign → ActionExecution → new Episode` 的执行桥已作为当前闭环的一部分被记录
- 这说明系统已经有“从推荐动作到执行记录”的结构化落点

3. 面向 v8 的程序化对象边界已经被收口

- `MechanismProgram` 合同已存在，但仍是 `draft`
- `CounterfactualScenario` 合同已存在，但仍是 `draft`
- `ExperimentDesign` 合同已存在，但仍是 `draft`
- `OutcomeRecord` 合同已存在，但仍是 `draft`

这类对象的意义不是证明“能力已完成”，而是说明当前主线已经把 v8 的关键接口**钉住了语义边界**。

### 3.2 先吸收的真实核心

因此，v12b 所说的“选择性吸收 v8”，本质上是先吸收以下三件事：

- 把 `ObservationModel` 作为 v8 的首个稳定落点
- 把 `MechanismProgram / CounterfactualScenario / ExperimentDesign` 先压成受约束合同，而不是自由叙事
- 把 `ActionExecution` 接成从未来实验层回流 Episode 的最小执行桥

这一步的意义，不是系统已经会稳定预测未来，而是系统已经开始为预测与干预能力准备**可审计接口**。

---

## 4. 已锚定能力 vs 仍处于预校准状态

### 4.1 已锚定能力

当前可以相对确定地说，主线已经锚定了以下能力边界：

1. 从“观测即事实”转向“观测是投影”

- 有 `ObservationModel`
- 有 `ObservationRecord → ObservationModel` 回溯链
- 有盲区、噪声、偏差的最小声明位置

2. 从“动作只是过程注释”转向“动作是闭环中的显式执行对象”

- 有 `ActionExecution`
- 有 source / target Episode 桥接语义
- 已为后续 prediction error 留出结构位置

3. 从“v8 只存在于设计史”转向“v8 的关键对象已有合同边界”

- 机制程序
- 反事实场景
- 实验设计
- 结果记录

这意味着系统已经不再停留于纯描述性重建的封闭世界，而是开始为“预测前写下结构化预期、执行后记录结构化反馈”预留统一语言。

### 4.2 仍处于预校准状态的部分

以下部分当前仍不能被写成既成事实：

1. `MechanismProgram` 的真实执行能力

- 合同还是 `draft`
- 文档明确写着“第一轮不要求完整执行引擎”
- 还没有证据表明 replay 已主要由程序对象驱动

2. `CounterfactualScenario` 的真实预测能力

- 合同还是 `draft`
- 文档允许“最小占位策略”生成预测轨迹
- 这说明当前对象边界先行，真实反事实生成仍待校准

3. `ExperimentDesign` 的信息增益与区分力计算

- 合同还是 `draft`
- 文档明确“不要求真实信息增益算法”
- 因而当前不能宣称系统已具备稳定主动学习能力

4. `OutcomeRecord` 到 `PredictionError` 的完整校准链

- `OutcomeRecord` 仍是 `draft`
- `PredictionError` 还没有当前合同作为稳定对象被收口
- v8 设计史中的“预测先于执行”“预测误差落盘”仍主要属于目标法律，不是已验证现实

5. v8 所需仿真环境

- 路线图明确要求离散事件仿真器 / MuJoCo / AI2-THOR 等验证环境
- 当前文档没有证据表明这类环境已成为日常主链路能力

因此，当前最准确的表述只能是：系统已经吸收了 v8 的**接口骨架与最小闭环锚点**，但尚未完成 v8 的**预测校准实装**。

---

## 5. 为什么先吸收“可执行 / 可校准子集”

原因不是保守，而是架构顺序必须正确。

### 5.1 先让对象有可审计边界，再谈世界引擎

如果没有：

- `ObservationRecord.observationModelId`
- `MechanismProgram` 的显式对象位
- `CounterfactualScenario` 的最小结构
- `ExperimentDesign` 的推荐动作位
- `ActionExecution` 的执行桥

那么后续所谓“预测”“反事实”“实验”都会退化成自由文本。

先吸收子集，是为了让这些未来能力进入合同、存储、审计、测试的共同边界。

### 5.2 先让未来能力可被证伪，再让它变强

当前最关键的不是让系统先看起来像世界引擎，而是让它未来一旦开始预测，就有明确的：

- 来源对象
- 执行动作
- 结果记录
- 回流 Episode

没有这些结构，再强的预测都无法进入可校准闭环。

### 5.3 先承认校准条件还没齐，再避免叙事超前

路线图已经明确：v8 需要仿真环境才能验证。

所以在仿真、回放、预测误差统计、信息增益评估尚未成为稳定现实前，最稳妥的设计史写法不是“v8 已经实现”，而是“v8 正在按可校准顺序被吸收”。

---

## 6. 边界如何表述：从描述性重建走向预测 / 干预世界引擎

v12b 的关键边界不是“有无 v8 文档”，而是主线正在跨过哪条架构分界。

这条分界可以写成：

```text
v7 主体：描述性重建世界
v8 目标：预测并干预世界
v12b 当前：只先吸收两者之间可被现有主线承受的过渡层
```

更具体地说：

### 6.1 描述性重建侧

已经较稳的是：

- Episode 采样
- ObservationRecord
- MechanismInstance / Reconstruction
- OntologyDelta
- proof-carrying 的回溯要求

这仍以“发生了什么、为什么会这样”为主。

### 6.2 预测 / 干预侧

真正高阶的 v8 目标是：

- 推断潜在状态
- 前向运行机制程序
- 生成反事实分叉
- 设计区分实验
- 记录预测误差
- 用误差校准本体与有效域

这一侧当前还没有足够多的实现与验证证据，不能写成已达成现实。

### 6.3 v12b 的真实位置

因此 v12b 代表的不是“v8 的缩减版”，而是：

> **把描述性重建系统，推进到一个已经具备预测 / 干预接口，但尚未宣称完整预测 / 干预能力的前沿状态。**

换句话说，当前系统正在从：

```text
descriptive reconstruction engine
```

向：

```text
predictive / interventional world engine
```

迁移，但只吸收其中已经有对象锚点、校准路线和测试前提的那一段。

---

## 7. 对 v8 的最终判断

v12b 不否定 v8，也不把 v8 降为“以后再说”。

v12b 对 v8 的判断是：

```text
v8 的方向是对的，且已经开始进入当前主线；
但当前主线吸收的不是“完整生成式世界引擎”，
而是其中可执行、可审计、未来可校准的先行层。
```

因此当前最准确的架构表述应是：

- 已进入当前现实的，是 v8 的观测投影化与最小实验闭环接口
- 尚未完成校准的，是 v8 的程序执行、反事实生成、信息增益计算与预测误差驱动的本体校准
- 当前边界应被理解为：从描述性重建，向预测 / 干预世界引擎的受约束迁移

这就是“选择性吸收 v8”的设计史含义。
