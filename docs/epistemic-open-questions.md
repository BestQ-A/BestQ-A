# 认识论开放问题

## 这篇文档是什么

记录当前认识论框架中**已识别但尚未解决**的问题。这些不是 bug，而是需要更多思考或经验积累才能回答的设计层问题。

---

## OQ-1: MechanismClass 的边界判定标准

**问题**：什么时候两个看起来相似的机制应该被归为同一个 `MechanismClass`，什么时候应该拆为两个？

**当前做法**：依赖人工审查和 `SplitClass` / `MergeClass` 提案。

**缺口**：没有形式化的"类边界判定标准"。当前依赖经验判断。

**可能方向**：
- 基于 `Episode` replay 一致性的量化指标
- 基于 `PredictionError` 分布的统计检验

---

## OQ-2: 推导链深度与可理解性的张力

**问题**：A1（推导链优先于结论）要求完整推导链。但推导链越长，人类可理解性越差。

**当前做法**：推导链完整记录，但 `summary-markdown-contract.md` 提供人类可读摘要。

**缺口**：摘要生成没有形式化的保真度保证。摘要可能丢失关键推导步骤。

**可能方向**：
- 定义"摘要保真度"指标
- 要求摘要携带到原始推导链的映射

---

## OQ-3: PredictionError 的积累阈值

**问题**：多少 `PredictionError` 足以触发本体升级？

**当前做法**：手动触发或 agent 主动发起。

**缺口**：没有量化的"积累阈值"。太敏感会导致过度升级，太迟钝会导致模型过时。

**可能方向**：
- 基于误差分布变化的 CUSUM 检测
- 基于 Episode 覆盖率的自适应阈值

---

## OQ-4: 反事实场景的可信度评估

**问题**：`CounterfactualScenario` 的预测轨迹有多可信？如何评估不曾发生的事情的预测质量？

**当前做法**：记录置信度字段，但没有校准机制。

**缺口**：置信度是主观标注，不是校准后的概率。

**可能方向**：
- 与历史上类似干预的实际结果对比
- 多个独立 agent 的预测一致性作为代理指标

---

## OQ-5: 本体演化的不可逆性风险

**问题**：A3 要求可逆建模，但某些本体变更（如 `MergeClass`）在实践中可能难以完全回滚，因为依赖该类的所有 Episode 都已经被重新绑定。

**当前做法**：`OntologyDelta` 记录变更，理论上可回滚。

**缺口**：实际回滚路径未被测试过。级联影响可能使回滚成本高到不可行。

**可能方向**：
- 引入"软回滚"（保留旧类但标记 deprecated，而非真正删除）
- 在 `OntologyDelta` 中记录影响范围，评估回滚成本

---

## OQ-6: 多 agent 并行时的本体一致性

**问题**：当多个 agent 同时生成 `OntologyUpdate` 提案时，如何保证最终本体的一致性？

**当前做法**：`ReviewDecision` 门控，顺序审批。

**缺口**：顺序审批在高并发时成为瓶颈。但并行审批可能导致冲突。

**可能方向**：
- 乐观并发 + 冲突检测 + 自动回滚
- 分域锁定（不同 `EntityClass` 域可并行升级）
