# 推荐下一个实验

## 核心理念

从已有实验结果中推断：哪个参数组合的信息增益最大——即做了这个实验后，对系统理解的提升最大。

## 执行步骤

### Step 1：获取当前知识状态

```json
mcp__causal-learner__get_stats
mcp__causal-learner__list_regulations {"limit": 50}
mcp__causal-learner__list_events {"limit": 100}
```

### Step 2：构建参数空间

从 events 的 facts 中提取：
- 所有出现过的参数名（pred="cause" 的 value 中的参数）
- 每个参数测试过的值列表
- 每个参数值对应的效果

### Step 3：信息增益分析

**3a. 不确定性最高的参数**
- 找出 regulations 中 contradictionRate 最高的 → 说明系统对该参数的理解最不确定
- 针对这些参数建议确认实验

**3b. 外推区域**
- 找出参数值只在小范围测试过的 → 建议扩大范围
- 例如 d_cp 只试了 3.2-3.8 → 建议试 2.8 或 4.2

**3c. 组合实验**
- 找出从未同时改变过的参数对 → 建议交互实验
- 如果 A 和 B 从未同时改过，A×B 的交互效应未知

**3d. 反事实验证**
- 找出只有正面（improved）证据没有反例的 regulations → 建议反向实验验证
- 例如"加大 X 总是好的" → 建议试试减小 X 确认是否总是差

### Step 4：排序推荐

按信息增益估计排序（高→低）：
1. 不确定性最高的 → 最需要确认
2. 外推区域 → 可能发现新规律
3. 组合实验 → 发现交互效应
4. 反事实验证 → 加固已有知识

### Step 5：输出建议

```
=== 下一个实验推荐 ===

基于 <N> 条观测和 <M> 条规律分析：

🥇 推荐 1（不确定性消除）:
   实验: <具体参数设置>
   原因: <param> 在现有数据中矛盾率 <X>%，需要确认
   预期: 确认或推翻规律 "<regulation 描述>"

🥈 推荐 2（外推探索）:
   实验: <具体参数设置>
   原因: <param> 只在 <range> 内测试过，<新值> 区域未知
   预期: 可能发现新的性能拐点

🥉 推荐 3（交互效应）:
   实验: 同时改 <param1>=<val> 和 <param2>=<val>
   原因: 两个参数从未同时改变，交互效应未知
   预期: 发现是否存在协同/对抗效应

[如果知识库太少]
⚠️ 当前只有 <N> 条观测，建议先多录入实验数据再分析。
   用 /learn-natural 快速录入已有实验结果。
```
